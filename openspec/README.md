# OpenSpec

This directory is the durable product and engineering specification for the lightweight intelligent chat system.

Future changes should read these files before editing code:

1. `openspec/project.md`
2. `openspec/specs/chat/spec.md`
3. The relevant change record under `openspec/changes/`

The specs use normative keywords:

- MUST: required behavior or constraint
- SHOULD: preferred behavior unless there is a strong local reason
- MAY: optional behavior

Secrets, API keys, local-only machine paths, and one-off test data MUST NOT be written into specs.
