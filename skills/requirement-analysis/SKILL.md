# Requirement Analysis

Use this skill when a user asks to clarify software requirements, design a feature, split work into phases, or turn product intent into implementation tasks.

## Workflow

1. Identify the user's goal and the smallest useful version.
2. Separate in-scope behavior from future enhancements.
3. Name assumptions explicitly.
4. Propose 2-3 implementation approaches when trade-offs matter.
5. Convert the selected approach into concrete backend, frontend, data, and validation tasks.

Keep the answer concise and practical. Prefer clear requirements over broad speculation.
